# 非サポート機能一覧

Tsuchinokoトランスパイラが現在サポートしていないPython機能の一覧です。

## 言語機能

- **`**kwargs`** (キーワード可変長引数)
- **デコレータ** (`@staticmethod`, `@dataclass` 以外)
- **ジェネレータ** (`yield` 文)
- **Async/Await**構文
- **コンテキストマネージャ** (`with` 文)
- **多重継承**
- **メタクラス**
- **Walrus演算子** (`:=`)

## 例外処理

- **try/except/finally** による特定の例外型捕捉
- **カスタム例外クラス**
- **例外連鎖** (`raise ... from ...`)

## クラス機能

- **クラス継承** (スライスのような基本クラスを除く)
- **プロパティ** (`@property`)
- **クラスメソッド** (`@classmethod`)
- **`__repr__`, `__str__`** およびその他のマジックメソッド
- **演算子オーバーロード** (`__add__`, `__eq__` など)
- **抽象基底クラス**

## 組み込み型

- **Set型** (`set[T]`)
- **Frozenset**
- **複素数**
- **Bytes/Bytearray**
- **Decimal**

## 標準ライブラリ / 外部ライブラリ

> [!NOTE]
> 多くの標準および外部ライブラリは、**常駐プロセス方式 (V1.2.0)** によりサポートされるようになりました。
> ただし、純粋なRustコードへの変換（Native Transpilation）はまだ対応していません。

### 非サポート (Native 変換)

以下は *純粋なRust* には変換できませんが、常駐プロセス方式では動作する可能性があります：

- **ファイルI/O** (`open()`, ファイル操作)
- **正規表現** (`re` モジュール)
- **日付/時間** (`datetime` モジュール)
- **コレクション** (`collections` モジュール - dict以外)
- **モジュールシステム** (相対インポートを含む複雑な複数ファイルプロジェクト)

### 常駐プロセス方式でサポート ✅

以前は非サポートでしたが、IPC経由で動作するようになったもの：

- **numpy**
- **pandas**
- **scipy**
- **ctypes** 依存ライブラリ

### 常駐ライブラリにおける非サポート構文

常駐プロセス方式でライブラリ自体は使用可能ですが、以下の構文は正しく変換できない場合があります：

- **外部型の型エイリアス**: `NDInt = npt.NDArray[np.int64]` (コンパイル時に外部型を解決できません)
- **高度な演算子オーバーロード**: `df[df["col"] > 5]` (Pandas のフィルタリング構文などは複雑なため変換困難です)
- **オブジェクト固有メソッドの型推論**: 複雑なオブジェクトを返すメソッドは、Rust側で型情報が失われる可能性があります (`-> _`)。

## 高度な機能

- **多重代入** (`a, b, c = 1, 2, 3` - インデックススワップのみ部分対応)
- **辞書内包表記** (`{k: v for k, v in ...}`)
- **セット内包表記** (`{x for x in ...}`)
- **Global/Nonlocal** 文

## ノート

ここに記載されている機能は将来のバージョンで追加される可能性があります。機能リクエストについてはGitHubリポジトリでIssueを作成してください。
